Etapa de Pruebas

Esta etapa de las pruebas de software va completamente de la mano con los temas de calidad de software, ya que por medio de dichas pruebas, se puede determinar que el software funciona como fue diseñado.

A través de las pruebas se pueden encontrar y mitigar los errores de software para ser corregidos antes de que el mismo llegue a su producción. Una de las características más estratégicas de las pruebas de Software es que se aplican de adentro hacia afuera.

Dichas pruebas se aplican de forma individual a cada módulo o subconjunto de programas, estas pruebas se llaman pruebas de unidad, seguida de esta encontramos las pruebas de integración, donde el foco de atención es el diseño y construcción de la arquitectura del software. Luego siguen las pruebas de validación, donde como su nombre lo dice se validan los requisitos establecidos como parte del análisis de requisitos del sistema que ha sido construido.

Finalmente se implementan las pruebas del sistema, las cuales se prueba el sistema como un todo, el software y otros elementos del sistema.

Para documentar las pruebas de Software, hay un formato con una estandarización internacional, llamado: IEEE 829.
